# DWCX 综合业务平台

双击 `启动DWCX综合业务平台.bat` 启动。

左侧导航包含：工作台、数据分析、身份证 OCR、优惠计算、日常总结、周末总结、手续费结算和历史数据。

## 模块接入方式

- 优惠计算：直接嵌入总程序，复用原脚本的规则与计算函数。
- 日常总结：后台调用原 `process_excel.py`，界面分别选择当日报价表、当日报价记录表和目标日期，再按车牌匹配生成结果。
- 手续费结算：分别选择手续费数据文件和单个结算模板，模板名称决定车险或非车业务类型，再按月份和渠道生成对应结算单。
- 身份证 OCR：由总程序启动原完整窗口，继续使用 `id_card` 数据库。
- 周末总结：选择询单/订单表、保单信息表和统计日期，生成周末总结、询单检查表、生效保单检查表及未匹配代理人检查表，并支持历史报告预览。
- 历史数据：显示 `历史数据` 目录中 ZIP 档案的文件清单，并支持按压缩包名称或档案内文件名查询。
- 数据分析：上传与样例表头一致的报价表、保单表，生成报价、保单、联合和数据质量分析。

## 数据分析维度

- 报价分析：订单状态漏斗、月度趋势、代理人/渠道、业务方案、业务员工作量、核保风险关键词、车牌地域、起保日期差异。
- 保单分析：保单状态、每日出单趋势、业务员绩效、出单员绩效、保费构成、车牌地域、订单保费区间。
- 联合分析：通过订单号计算关联覆盖率，并按渠道、业务方案、报价状态分析关联生效订单、保费与录单耗时。
- 数据质量：必要字段完整率、缺失/占位值、唯一值、重复订单/投保单号及车架号规范性。

“联合分析”的关联覆盖率会受到两份文件日期范围影响，不应直接视为完整业务转化率。

## 界面与架构

- 视觉采用黑、白、灰三色：黑色导航栏、白色功能卡片和浅灰工作区。
- 页面共享统一的按钮、输入框、表格、日志区和状态提示样式。
- `BackendService` 统一负责模块缓存、数据库连接、命令执行和子进程管理。
- 耗时的 Excel 与数据库操作使用后台线程，避免主界面卡死。
- 数据分析、总结生成、结算、OCR、数据库检查和历史数据刷新统一显示 Loading 动画与当前任务提示。
- OCR 窗口采用单实例管理，重复点击不会启动多个 OCR 进程。
- 数据库参数可通过 `DWCX_DB_HOST`、`DWCX_DB_PORT`、`DWCX_DB_USER`、`DWCX_DB_PASSWORD`、`DWCX_DB_NAME` 环境变量覆盖。

## 数据库

- 主机：`localhost`
- 端口：`3306`
- 数据库：`id_card`
- 用户：`root`

密码沿用本机已配置值，不写入本说明文件。
